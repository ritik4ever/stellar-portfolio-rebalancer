# Architecture Decision Records (ADRs)

This directory contains records of significant architectural decisions made for the Stellar Portfolio Rebalancer project.

## What is an ADR?

An Architecture Decision Record (ADR) is a document that captures an important architectural decision made along with its context and consequences. For more background on ADRs, see [Documenting Architecture Decisions](https://cognitect.com/blog/2011/11/15/documenting-architecture-decisions.html).

## Workflow

1. Use the [ADR template](template.md) to draft a new decision.
2. Number the file sequentially (e.g., `0002-my-new-decision.md`).
3. Open a Pull Request for discussion.
4. Once merged, the ADR is marked as **Accepted**.
