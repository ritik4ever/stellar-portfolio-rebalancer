# Internationalization (i18n) Preparation

This document outlines the strategy for adding internationalization support to the Stellar Portfolio Rebalancer frontend.

## Current State

The UI copy is currently hardcoded in English across all components. No i18n library is configured.

## Recommended Approach

### Library: react-i18next

[react-i18next](https://react.i18next.com/) is the most popular React i18n library and integrates well with TypeScript.

### File Structure

```
frontend/
  public/
    locales/
      en/
        translation.json
        common.json
        portfolio.json
        dashboard.json
        errors.json
      es/
        translation.json
        common.json
        portfolio.json
        dashboard.json
        errors.json
      fr/
        translation.json
        ...
  src/
    i18n/
      config.ts          # i18n initialization
      types.ts           # TypeScript type definitions
      resources.ts       # Resource loader
```

### Implementation Steps

1. **Install dependencies**
```bash
npm install react-i18next i18next i18next-browser-languagedetector
```

2. **Initialize i18n**
```typescript
// src/i18n/config.ts
import i18n from "i18next";
import { initReactI18next } from "react-i18next";
import LanguageDetector from "i18next-browser-languagedetector";

i18n
  .use(LanguageDetector)
  .use(initReactI18next)
  .init({
    fallbackLng: "en",
    interpolation: { escapeValue: false },
    resources: {
      en: { translation: {} },
      es: { translation: {} },
    },
  });

export default i18n;
```

3. **Extract hardcoded strings** into translation JSON files:
```json
// public/locales/en/common.json
{
  "app.title": "Stellar Portfolio Rebalancer",
  "nav.overview": "Overview",
  "nav.analytics": "Analytics",
  "nav.settings": "Settings",
  "button.connect": "Connect Wallet",
  "button.create": "Create Portfolio",
  "state.loading": "Loading...",
  "state.error": "An error occurred",
  "state.empty": "No portfolios yet"
}
```

4. **Replace hardcoded strings** with `useTranslation()` hook:
```tsx
import { useTranslation } from "react-i18next";

function Dashboard() {
  const { t } = useTranslation();
  return <h1>{t("app.title")}</h1>;
}
```

## File Size Budget

- Each locale translation file should stay under 50KB
- Lazy load locale files to avoid bundling all languages
- Use namespaces to split by feature area

## TypeScript Integration

Create strongly-typed `t()` function:

```typescript
// src/i18n/types.ts
import type common from "../../public/locales/en/common.json";

export type TranslationKeys = {
  [K in keyof typeof common]: string;
};
```
