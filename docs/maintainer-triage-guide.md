# Maintainer Triage Guide

## Issue Triage Process

### New Issues
1. **Acknowledge**: Reply within 24 hours acknowledging receipt
2. **Categorize**: Apply labels (bug, feature, docs, question, etc.)
3. **Prioritize**: Assign priority based on impact and urgency
4. **Bounty Assignment**: If eligible, tag with Stellar Wave bounty label

### Bug Reports
- Verify reproduction steps
- Check for duplicate reports
- Add severity label (critical, major, minor)
- Assign to appropriate milestone

### Feature Requests
- Evaluate alignment with project roadmap
- Request community feedback if needed
- Tag as enhancement for bounty consideration

## Pull Request Triage Process

### PR Review Checklist
1. **CI Status**: All checks must pass
2. **Code Quality**: Follows project conventions
3. **Test Coverage**: New code includes tests
4. **Documentation**: Relevant docs are updated
5. **Closes Issue**: PR body references the issue number

### Merge Guidelines
- Feature branches require at least one maintainer approval
- Hotfixes can be merged with expedited review
- Breaking changes require a discussion issue first

## Labels Reference

| Label | Description |
|-------|-------------|
| bug | Something isn't working |
| enhancement | New feature or request |
| docs | Documentation changes |
| good first issue | Good for newcomers |
| Stellar Wave | Bounty-eligible issue |
